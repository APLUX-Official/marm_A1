# 舵机角度偏差补偿功能说明

## 概述

由于舵机安装可能存在偏差，此功能通过YAML配置文件对各个舵机的实际控制角度进行补偿。

## 配置文件位置

```
arm_controller/config/servo_calibration.yaml
```

## 配置文件格式

```yaml
servo_offsets:
  servo_0: 0.0  # joint1 角度偏差（度）
  servo_1: 0.0  # joint2 角度偏差（度）
  servo_2: 0.0  # joint3 角度偏差（度）
  servo_3: 0.0  # joint4 角度偏差（度）
  servo_4: 0.0  # joint5 角度偏差（度）
  servo_5: 0.0  # joint6 夹爪角度偏差（度）
```

## 使用方法

### 1. 标定舵机偏差

1. 通过ROS2话题发送一个标准角度（例如0度）给舵机
2. 测量舵机实际到达的角度
3. 计算偏差值：
   - 如果舵机实际角度**小于**命令角度（例如命令0度，实际-5度），说明需要**增加**角度，设置正值（+5）
   - 如果舵机实际角度**大于**命令角度（例如命令0度，实际+5度），说明需要**减少**角度，设置负值（-5）

### 2. 修改配置文件

编辑 `config/servo_calibration.yaml` 文件，设置各个舵机的偏差值。

例如，如果joint2（servo_1）安装偏差导致实际角度总是比命令角度小3度：

```yaml
servo_offsets:
  servo_0: 0.0
  servo_1: 3.0  # 补偿+3度
  servo_2: 0.0
  servo_3: 0.0
  servo_4: 0.0
  servo_5: 0.0
```

### 3. 重新编译和运行

```bash
# 在工作空间根目录
cd ~/01-program/05-mbot-marm/marm_ws

# 重新编译
colcon build --packages-select arm_controller

# 刷新环境变量
source install/setup.bash

# 运行节点
ros2 run arm_controller arm_controller
```

## 工作原理

程序在启动时会自动加载配置文件，并在每次发送舵机控制指令前自动应用角度补偿：

```
实际发送角度 = 命令角度 + 偏差值
```

这样确保舵机能够到达预期的实际位置。

## 注意事项

1. **角度范围限制**：补偿后的角度仍需在舵机允许范围内（-135° ~ +135°）
2. **配置修改生效**：修改配置文件后需要重启节点才能生效
3. **备份建议**：标定完成后建议备份配置文件
4. **精度**：配置精度到小数点后一位即可（如：3.5）

## 故障排除

### 配置文件未加载

如果日志显示 "Servo calibration file not found"，检查：
- 配置文件是否在正确位置
- 是否执行了 `colcon build`
- 是否执行了 `source install/setup.bash`

### 补偿效果不佳

1. 重新测量舵机实际角度
2. 检查是否选择了正确的偏差方向（正/负）
3. 逐步调整偏差值，每次调整2-3度
4. 在多个角度点进行测试验证

## 示例

假设标定结果如下：

| 舵机ID | 命令角度 | 实际角度 | 计算偏差 | 配置值 |
|--------|---------|---------|---------|--------|
| servo_0 | 0° | -2° | 需要+2° | 2.0 |
| servo_1 | 0° | +3° | 需要-3° | -3.0 |
| servo_2 | 0° | 0° | 无偏差 | 0.0 |
| servo_3 | 0° | -1° | 需要+1° | 1.0 |
| servo_4 | 0° | 0° | 无偏差 | 0.0 |
| servo_5 | 0° | +5° | 需要-5° | -5.0 |

对应的配置文件：

```yaml
servo_offsets:
  servo_0: 2.0
  servo_1: -3.0
  servo_2: 0.0
  servo_3: 1.0
  servo_4: 0.0
  servo_5: -5.0
```
