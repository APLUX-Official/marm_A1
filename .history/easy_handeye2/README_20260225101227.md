# Easy Handeye2

ROS2 Humble 版本的手眼标定工具包（C++ 实现）

## 概述

Easy Handeye2 是一个用于机器人手眼标定的 ROS2 包，提供：

- **采样**：通过 TF 采集机器人位姿和相机跟踪数据
- **计算**：使用 OpenCV 库实现多种标定算法
- **存储**：将标定结果保存为 YAML 文件
- **发布**：将标定结果作为 TF 变换发布

支持两种标定模式：
- **Eye-in-Hand（眼在手上）**：相机安装在机械臂末端
- **Eye-on-Base（眼在底座）**：相机固定在机器人外部

**所有参数均通过 YAML 配置文件设置，无需通过服务修改。**

## 配置

所有配置参数都通过 YAML 文件设置。参考 `config/` 目录下的配置示例：

- `calibration_params.yaml` - 基本配置模板
- `eye_in_hand_example.yaml` - Eye-in-Hand 配置示例
- `eye_on_base_example.yaml` - Eye-on-Base 配置示例

### 配置参数说明

```yaml
calibration:
  name: "my_calibration"           # 标定名称
  eye_on_hand: false               # true=眼在手上, false=眼在底座
  robot_base_frame: "base_link"    # 机器人基座坐标系
  robot_effector_frame: "tool0"    # 机器人末端坐标系
  tracking_base_frame: "camera_link"  # 相机坐标系
  tracking_marker_frame: "marker"     # 标记物坐标系

algorithm:
  name: "Tsai-Lenz"                # 标定算法

publisher:
  publish_static_tf: true          # 发布静态TF
  publish_rate: 50.0               # 发布频率（动态TF时使用）
```

## 使用方法

### 1. 启动标定服务

```bash
# 使用默认配置
ros2 launch easy_handeye2 calibrate.launch.py

# 使用自定义配置
ros2 launch easy_handeye2 calibrate.launch.py config_file:=/path/to/config.yaml
```

### 2. 调用服务进行标定

```bash
# 采集样本（移动机器人到不同位置后执行）
ros2 service call /handeye_calibration_server/take_sample easy_handeye2_msgs/srv/TakeSample

# 计算标定
ros2 service call /handeye_calibration_server/compute_calibration easy_handeye2_msgs/srv/ComputeCalibration

# 保存标定结果
ros2 service call /handeye_calibration_server/save_calibration easy_handeye2_msgs/srv/SaveCalibration "{filepath: ''}"
```

### 3. 发布标定结果

```bash
# 发布已保存的标定结果
ros2 launch easy_handeye2 publish.launch.py calibration_name:=my_calibration

# 指定标定文件
ros2 launch easy_handeye2 publish.launch.py calibration_file:=/path/to/calibration.yaml
```

## 支持的算法

| 算法 | 说明 |
|------|------|
| Tsai-Lenz | 经典算法，默认推荐 |
| Park | 噪声鲁棒性好 |
| Horaud | 旋转和平移分离求解 |
| Andreff | 基于运动学约束 |
| Daniilidis | 使用双四元数表示 |
| Constrained | 约束优化，用于有限自由度场景 |

## 服务接口

| 服务名 | 类型 | 说明 |
|--------|------|------|
| ~/take_sample | TakeSample | 采集一个样本 |
| ~/compute_calibration | ComputeCalibration | 计算标定 |
| ~/save_calibration | SaveCalibration | 保存标定结果 |

算法、坐标系等参数均通过 YAML 配置文件设置，不提供运行时修改服务。

## 标定结果文件

标定结果保存在 `~/.ros/easy_handeye2/` 目录下，格式为 YAML：

```yaml
parameters:
  name: my_calibration
  eye_on_hand: false
  robot_base_frame: base_link
  robot_effector_frame: tool0
  tracking_base_frame: camera_link
  tracking_marker_frame: marker
transformation:
  x: 0.5
  y: 0.1
  z: 0.8
  qx: 0.0
  qy: 0.0
  qz: 0.0
  qw: 1.0
```

## 标定流程建议

1. 确保机器人和相机的 TF 树正确发布
2. 根据实际情况修改配置文件中的坐标系名称和算法
3. 采集 10-20 个样本（覆盖不同位置和角度）
4. 计算标定并检查结果
5. 保存标定结果
6. 在应用中启动发布节点使用标定结果

## 样本数据自动保存功能

### 功能说明

- **自动保存**：每次执行 `take_sample` 后，数据自动保存到 YAML 文件
- **自动加载**：服务启动时自动加载之前保存的样本数据
- **文件位置**：`~/.ros/easy_handeye2/samples_<calibration_name>.yaml`

### 清除样本数据

```bash
rm ~/.ros/easy_handeye2/samples_<calibration_name>.yaml
```
